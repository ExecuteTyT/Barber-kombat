# Модуль Barber Kombat

## Назначение

Ежедневные соревнования между барберами одного филиала с автоматическим расчётом рейтинга в реальном времени и денежными призами.

## Алгоритм расчёта рейтинга

### Шаг 1: Сбор сырых данных

Для каждого барбера, работающего в текущую смену, собираются данные за текущий день:

```python
# Из таблицы visits WHERE branch_id = X AND date = today AND status = 'completed'
raw_data = {
    "barber_id": uuid,
    "revenue": sum(visit.revenue),              # Общая выручка (услуги + товары)
    "visits": [...],                              # Список визитов для расчёта ЧС
    "products_count": sum(visit.products_count),  # Кол-во товаров (штуки)
    "extras_count": sum(visit.extras_count),      # Кол-во допов (штуки)
    "reviews_avg": avg(review.rating),            # Средняя оценка (если есть)
}
```

### Шаг 2: Расчёт ЧС (Чек/Стрижка)

Для каждого визита:
```
ЧС_визита = Сумма_услуг_визита / Цена_стрижки_мастера
```

Средний ЧС за день:
```
ЧС_дня = Сумма_ЧС_всех_визитов / Количество_визитов
```

Пример:
- Барбер с ценой стрижки 1600₽
- Визит 1: стрижка (1600) + борода (900) = 2500 → ЧС = 2500/1600 = 1.56
- Визит 2: только стрижка = 1600 → ЧС = 1600/1600 = 1.00
- Средний ЧС = (1.56 + 1.00) / 2 = 1.28

### Шаг 3: Нормализация

По каждому параметру определяется лидер (максимальное значение = 100%). Остальные получают процент от лидера.

```python
def normalize(values: list[float]) -> list[float]:
    max_val = max(values)
    if max_val == 0:
        return [0.0] * len(values)
    return [(v / max_val) * 100 for v in values]
```

Пример по выручке:
- Павел: 13,500₽ → 100%
- Лев: 12,300₽ → 91.1%
- Марк: 6,500₽ → 48.1%

### Шаг 4: Взвешенная сумма

```python
total_score = (
    revenue_score * weights.revenue / 100 +
    cs_score * weights.cs / 100 +
    products_score * weights.products / 100 +
    extras_score * weights.extras / 100 +
    reviews_score * weights.reviews / 100
)
```

Веса по умолчанию: Выручка 20%, ЧС 20%, Товары 25%, Допы 25%, Отзывы 10%.

### Шаг 5: Ранжирование

Барберы сортируются по total_score (DESC). Присваиваются места: 1, 2, 3...
При равных баллах — выше тот, у кого больше выручка.

## Призовой фонд

### Формирование

Призовой фонд = процент от накопительной выручки филиала с 1-го числа месяца.

```python
branch_monthly_revenue = sum(visits.revenue WHERE branch_id = X AND date >= first_day_of_month)

prize_fund = {
    "gold": branch_monthly_revenue * config.prize_gold_pct / 100,    # 0.5%
    "silver": branch_monthly_revenue * config.prize_silver_pct / 100, # 0.3%
    "bronze": branch_monthly_revenue * config.prize_bronze_pct / 100, # 0.1%
}
```

### Определение победителя месяца

По итогам месяца — не по среднему рейтингу, а по количеству побед (дней на 1-м месте).
При равенстве побед — у кого больше суммарный рейтинг за месяц.

```python
monthly_standings = (
    SELECT barber_id, COUNT(*) as wins, SUM(total_score) as total
    FROM daily_ratings
    WHERE branch_id = X AND date >= first_day AND date <= last_day AND rank = 1
    GROUP BY barber_id
    ORDER BY wins DESC, total DESC
)
```

## Граничные случаи

1. **Барбер без визитов за день:** все показатели = 0, рейтинг = 0, участвует в таблице
2. **Один барбер в смене:** автоматически получает 100% по всем параметрам, 1-е место
3. **Все барберы с нулевой выручкой:** все получают 0%, рейтинг определяется другими параметрами
4. **Барбер начал работать в середине дня:** считается от момента первого визита
5. **Отменённый визит:** не учитывается в расчётах (status != 'completed')
6. **Смена грейда/цены в середине месяца:** ЧС рассчитывается по текущей цене
