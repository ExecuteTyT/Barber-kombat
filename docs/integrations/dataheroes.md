# Интеграция DataHeroes

Статус: **модель сосуществования принята.** API у DataHeroes нет (проверено) →
интеграция «pull/push» невозможна; работаем по модели №1. Канал отзывов отдан DataHeroes —
наш авто-запрос отключён флагом `review_requests_enabled=False`.
Дата: 2026-05-31.

## 1. Что такое DataHeroes

[DataHeroes](https://app.dataheroes.pro) — внешний SaaS для бьюти-индустрии (салоны,
барбершопы, клиники). Две основные функции:

1. **WhatsApp-бот** — автоматические сообщения клиентам: запись, напоминания о визите,
   возврат («реактивация»), **запрос отзыва после визита**.
2. **Аналитика** — сегментация клиентов, KPI сотрудников, выручка, конверсия записей,
   ретеншн, частота визитов, средний чек; предиктивная модель (вероятность возврата/оттока)
   и what-if по бизнес-политикам.

Подключается к CRM (40+ систем, **в т.ч. YClients**, Altegio, 1С). Аккредитован в реестре
российского ПО. Тариф от ~2 743 ₽/мес, 14 дней бесплатно.

## 2. Ключевой архитектурный факт

**Интеграция идёт через YClients (подтверждено заказчиком).** DataHeroes подключается к тому
же YClients, что и MAKON. Это не «плагин» внутрь нашего бэкенда — это самостоятельный сервис,
который берёт данные напрямую из YClients салона. Отдельного API DataHeroes для нас нет —
обе системы работают как параллельные потребители одного YClients.

```
                 ┌────────────► MAKON (sync → PostgreSQL → рейтинги/ПВР/отчёты)
   YClients API ─┤
                 └────────────► DataHeroes (своя БД → дашборды + WhatsApp-бот)
```

Из этого следуют и возможности, и главный риск (раздел 4).

## 3. Возможные модели интеграции

Выбор зависит от того, есть ли у DataHeroes публичный API (см. «Открытые вопросы»).

| # | Модель | Когда применима | Изменения в коде MAKON |
|---|--------|-----------------|------------------------|
| 1 | **Сосуществование через YClients** (наиболее вероятно) | Всегда — DataHeroes просто подключают к YClients салона | Минимальные: разграничить зоны ответственности, отключить наш авто-запрос отзывов, если канал отдаём DataHeroes |
| 2 | **Pull из DataHeroes API** | Если у DataHeroes есть API/выгрузка | Новый `app/integrations/dataheroes/client.py` + поля в `Settings`; тянем предиктивную аналитику/сегменты в owner-дашборд |
| 3 | **Push событий в DataHeroes** | Если DataHeroes принимает входящие события/вебхуки | Эмитим наши события (visit.synced, review.submitted, pvr.threshold) — точки уже есть: Redis Pub/Sub `ws:org:*`, хуки после sync/review |

Рекомендация: начать с модели №1 (она работает без API), а №2/№3 добавлять, только если
DataHeroes предоставит интерфейс.

### Образец для модели №2/№3 (если будет API)
Конфиг по образцу существующих интеграций ([config.py](../../backend/app/config.py) — поля
YClients/WhatsApp):
```python
dataheroes_api_url: str = ""
dataheroes_api_key: str = ""
dataheroes_enabled: bool = False
```
Клиент — по образцу [whatsapp/client.py](../../backend/app/integrations/whatsapp/client.py)
(httpx, structlog, не валить основной поток при ошибке отправки).

## 4. ⚠️ Критический конфликт: WhatsApp-запросы отзывов

**И MAKON, и DataHeroes умеют слать клиенту WhatsApp-запрос отзыва после визита.** Если оба
подключены к одному YClients — клиент получит **два сообщения**.

- Наша реализация: [review_tasks.py](../../backend/app/tasks/review_tasks.py)
  (`send_review_request`, задержка `review_request_delay_minutes`),
  [whatsapp/client.py](../../backend/app/integrations/whatsapp/client.py) (GreenAPI),
  флаг `Visit.review_request_sent`.

**Решение (принято):** канал отзывов у **DataHeroes**. У нас он отключён:
- Настройка `review_requests_enabled` ([config.py](../../backend/app/config.py)), **по умолчанию `False`**.
- Планировщик в [webhook_tasks.py](../../backend/app/tasks/webhook_tasks.py) не ставит задачу
  `send_review_request`, пока флаг выключен → дублей WhatsApp нет.
- У нас остаётся только **приём/обработка** отзывов (если придут) и геймификация/ПВР.
- Включить обратно (если MAKON снова возьмёт канал): `REVIEW_REQUESTS_ENABLED=true` в env.

## 5. Что можно отслеживать (фокус: отзывы и взаимодействие с клиентом)

Ниже — что уже есть в данных MAKON, и что DataHeroes добавляет сверху.

### 5.1 Отзывы (модель [review.py](../../backend/app/models/review.py))
| Метрика | Источник | Примечание |
|---------|----------|------------|
| Распределение оценок 1–5 | `Review.rating` | По барберу/филиалу/периоду (индексы `barber_created`, `branch_created`) |
| Доля негатива | `rating ≤ 3` | Порог `_NEGATIVE_THRESHOLD = 3` |
| **SLA обработки негатива** | `processed_at − created_at` | Время реакции менеджера |
| Просрочка обработки | > 2 ч | `_OVERDUE_HOURS`, задача `check_unprocessed_reviews` |
| Воронка отзыва | визит завершён → `review_request_sent` → отзыв создан | См. пробелы (5.3) |
| Источник отзыва | `Review.source` (form/internal) | |
| Авто-обработка позитива | оценки 4–5 → `PROCESSED` автоматически | |

### 5.2 Взаимодействие с клиентом (модель [client.py](../../backend/app/models/client.py))
| Метрика | Источник |
|---------|----------|
| Повторность / ретеншн | `total_visits`, `first_visit_at`, `last_visit_at`; new vs returning, retention % ([reports.py](../../backend/app/services/reports.py) `generate_clients_report`) |
| Средний чек по типу клиента | `avg_check_new` / `avg_check_returning` |
| Частота и давность визитов (recency) | `last_visit_at`, `total_visits` |

### 5.3 Что добавляет DataHeroes сверх MAKON
- **Предиктивные сегменты**: вероятность возврата / риск оттока по клиенту.
- **What-if** по политикам (скидки, акции) на исторических данных.
- **Авто-сегментация** + готовые WhatsApp-сценарии (booking / reminder / retention / review).
- Конверсия записей, анализ незавершённых записей.

## 6. Пробелы трекинга в MAKON (мелкие доработки, отдельной задачей)
Нужны, если хотим полную воронку «визит → отзыв» у себя (а не только в DataHeroes):
- **Таймстамп отправки** запроса отзыва (сейчас только bool `Visit.review_request_sent`).
- **Клик/открытие ссылки** отзыва → конверсия «запрос → отзыв», время «визит → отзыв».
- **NPS-классификация** (промоутер 9–10 / нейтрал / детрактор) поверх оценок 1–5.

## 7. Открытые вопросы

Решено:
- ✅ **API DataHeroes** — публичного API/вебхуков/выгрузки нет (проверены `/api`, `/api/docs`,
  `/developers`, `/integrations` → 404 или SPA без Swagger; поиск тоже ничего не дал; PyPI-пакет
  `dataheroes` — другой продукт, ML-библиотека). Вывод: только модель №1.
- ✅ **Канал WhatsApp-отзывов** — у DataHeroes; наш отключён (раздел 4).

Осталось (опционально, на будущее):
- Нужны ли предиктивные данные DataHeroes **внутри** owner-дашборда MAKON? Без API это
  возможно только ручным экспортом/скриншотами — то есть практически нет. Если критично —
  запросить у DataHeroes индивидуальную выгрузку/доступ к их БД.
- Закрывать ли пробелы трекинга из раздела 6 (таймстамп отправки, клик по ссылке, NPS) —
  только если часть воронки отзывов хотим считать у себя, а не полагаться на DataHeroes.
